# Step 06: Define Integration, Data, and Test Boundaries

## Purpose

Show how the components interact, where data crosses boundaries, and what must be validated in real integration.

## Entry Criteria

- Step 05 exit criteria are met

## Actions

1. Define the major integration points.
2. Describe the key data or control flows between components.
3. Identify test boundaries and where real integration must be validated.
4. Create a simple architecture diagram that shows:
   - components
   - boundaries
   - main flows
   - test boundaries
5. Keep the flows implementation-relevant and easy to read.

## Expected Outputs

- integration-point summary
- data/control-flow summary
- test-boundary summary
- architecture diagram

## Exit Criteria

- the system interactions are explicit
- the main flows are understandable
- the test boundaries are visible
- the diagram is clear enough to guide implementation and review

## Proceed Condition

Proceed to Step 07 once the architecture interactions and test boundaries are documented clearly.
