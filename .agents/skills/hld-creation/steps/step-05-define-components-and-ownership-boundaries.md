# Step 05: Define Components and Ownership Boundaries

## Purpose

Break the architecture strategy into explicit components with clear ownership.

## Entry Criteria

- Step 04 exit criteria are met

## Actions

1. Identify the components required by the chosen architecture strategy.
2. Define the responsibility of each component.
3. Define what each component must not own.
4. Make ownership boundaries explicit.
5. Check the component model against the HLD type and source drivers.

## Expected Outputs

- component list
- component responsibilities
- ownership boundaries

## Exit Criteria

- each component has a clear role
- overlap and ambiguity are minimized
- the component model fits the architecture strategy

## Proceed Condition

Proceed to Step 06 once the component and ownership model is clear.
