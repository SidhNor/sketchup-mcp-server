# Step 09: Finalize and Persist HLD

## Purpose

Finalize the HLD as a usable high-level design reference for downstream planning and implementation.

## Entry Criteria

- Step 08 exit criteria are met

## Actions

1. Create or update the HLD using `templates/hld-template.md`.
2. Persist it as `specifications/hlds/hld-<slug>.md`.
3. Apply any corrections from validation.
4. Confirm the HLD type and scope still match the final document.
5. Summarize what was created or updated.

## Expected Outputs

- final HLD file
- completion summary

## Exit Criteria

- the HLD is stored in `specifications/hlds/`
- the HLD follows the required section structure
- the HLD scope is coherent
- opened questions capture the remaining unresolved architecture work

## Completion Check

Before finishing, confirm the HLD can guide detailed planning without forcing readers to reverse-engineer the intended architecture shape.
