# Step 02: Classify HLD Type and Document Boundaries

## Purpose

Set the document posture early so platform and capability concerns do not get mixed.

## Entry Criteria

- Step 01 exit criteria are met
- the user has confirmed or refined the architectural mandate

## Actions

1. Classify the HLD as `platform` or `capability`.
2. Define what belongs in this HLD.
3. Define what belongs in PRDs, other HLDs, or tasks instead.
4. Capture the major architectural goals and constraints for this HLD type.
5. Note any mixed-scope risks.

## Expected Outputs

- HLD type classification
- in-scope boundary
- out-of-scope boundary
- architectural goals and constraints
- mixed-scope risk list

## Exit Criteria

- the HLD type is explicit
- the document boundary is clear
- scope separation from PRDs and tasks is explicit

## Confirmation Gate

Present the HLD type and boundary summary, then ask the user for confirmation before proceeding to Step 03.
