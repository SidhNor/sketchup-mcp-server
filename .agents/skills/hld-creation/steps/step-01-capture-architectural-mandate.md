# Step 01: Capture Architectural Mandate

## Purpose

Define what architectural problem the HLD must solve based on source PRDs or platform needs.

## Entry Criteria

- a source PRD, platform need, guide, or user request exists

## Actions

1. Identify the source PRD obligations or platform needs that drive the HLD.
2. Summarize the architectural question the document must answer.
3. Capture the source documents that constrain the HLD.
4. Note ambiguities in the source mandate.
5. Avoid turning the step into a product summary.

## Expected Outputs

- architectural mandate summary
- source-driver list
- source-document list
- ambiguity list

## Exit Criteria

- the HLD has a clear architectural mandate
- the source drivers are explicit
- ambiguities that affect the architecture are visible

## Confirmation Gate

Present the architectural mandate and source drivers, then ask the user for confirmation before proceeding to Step 02.
