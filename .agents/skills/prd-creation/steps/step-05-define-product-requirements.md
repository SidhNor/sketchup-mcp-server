# Step 05: Define Product Requirements

## Purpose

Define the required product behaviors without slipping into architecture or implementation design.

## Entry Criteria

- Step 04 exit criteria are met

## Actions

1. Draft the functional requirements table.
2. Draft the non-functional requirements.
3. Prioritize the functional requirements.
4. Check each functional requirement against domain rules.
5. Flag any requirement that conflicts with domain rules.
6. Remove any requirement that is actually an implementation choice rather than a product need.

## Expected Outputs

- functional requirements table
- non-functional requirements
- requirement priorities
- domain-conflict notes

## Exit Criteria

- the PRD has a clear product requirement surface
- requirements are user-facing and implementation-agnostic
- domain conflicts are flagged explicitly

## Proceed Condition

Proceed to Step 06 once the product requirements are complete enough to bound the PRD.
