# Step 03: Align Domain Rules and Terminology

## Purpose

Make sure the PRD uses the right domain concepts and does not drift from core business rules.

## Entry Criteria

- Step 02 exit criteria are met
- the user has confirmed or refined the user problem and target users

## Actions

1. Read the relevant domain analysis when available.
2. Identify the domain entities, actions, states, and business rules that constrain the PRD.
3. Capture the canonical terminology the PRD must use.
4. Identify any product expectations that conflict with the current domain model.
5. Note domain gaps that will need to be carried as opened questions.

## Expected Outputs

- relevant domain-rule summary
- terminology conventions
- product/domain conflict notes
- domain-gap list

## Exit Criteria

- the PRD is grounded in the relevant domain rules
- canonical terms are identified
- domain conflicts are visible rather than hidden

## Confirmation Gate

Present the domain alignment summary and ask the user for confirmation before proceeding to Step 04.
