# Step 04: Shape Outcomes, KPIs, and Flows

## Purpose

Turn the product slice into measurable outcomes and concrete user flows.

## Entry Criteria

- Step 03 exit criteria are met
- the user has confirmed or refined the domain framing

## Actions

1. Define the product goals.
2. Identify success metrics and candidate KPIs.
3. Draft representative user flows and scenarios.
4. Check that the flows and metrics align with the user problem and domain constraints.
5. Capture baseline gaps and assumptions where measurement does not yet exist.

## Expected Outputs

- goals list
- draft success metrics
- primary and secondary KPI candidates
- user flows and scenarios
- metric assumptions

## Exit Criteria

- the product outcomes are measurable enough to frame the PRD
- user flows are concrete enough to drive requirements
- KPI candidates are explicit

## Confirmation Gate

Present the goals, KPI choices, and user flows, then ask the user for confirmation before proceeding to Step 05.
