# Step 06: Define Scope Boundaries and Delivery Risks

## Purpose

Complete the PRD by bounding the slice and documenting product-level risks and dependencies.

## Entry Criteria

- Step 05 exit criteria are met

## Actions

1. Define constraints.
2. Define out-of-scope items explicitly.
3. Capture opened questions.
4. Create the risks table.
5. Document dependencies.
6. Check that these sections support product scoping rather than architecture design.

## Expected Outputs

- constraints
- out-of-scope list
- opened questions
- risk table
- dependency list

## Exit Criteria

- the PRD scope is bounded clearly
- risks and dependencies are explicit
- exclusions prevent scope drift

## Proceed Condition

Proceed to Step 07 once the PRD has a complete product boundary.
