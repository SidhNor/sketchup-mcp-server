# Step 01: Define Product Slice

## Purpose

Determine the product slice boundary before writing a PRD.

## Entry Criteria

- a source guide, brief, or user request exists

## Actions

1. Identify the candidate product capability or capabilities in the source material.
2. Decide whether the work should be captured as one PRD or multiple PRDs.
3. Define the slice boundary for each proposed PRD.
4. Note overlaps, coupling, and reasons for any split.
5. Capture missing information that affects slice definition.

## Expected Outputs

- proposed PRD slice list
- slice-boundary summary
- split rationale
- ambiguity list

## Exit Criteria

- the PRD split is explicit
- each slice is coherent enough to support one product document
- ambiguities that affect the split are visible

## Confirmation Gate

Present the proposed PRD slice or slice set, then ask the user for confirmation before proceeding to Step 02.
