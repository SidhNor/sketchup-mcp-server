# Step 02: Capture User Problem and Target Users

## Purpose

Anchor the PRD in user pain, workflow friction, and the users affected by the slice.

## Entry Criteria

- Step 01 exit criteria are met
- the user has confirmed or refined the PRD slice

## Actions

1. Describe the user problem in workflow terms.
2. Identify the target users or user roles affected by the problem.
3. Capture why the problem matters to those users.
4. Note user-facing failure modes or workflow pain points.
5. Avoid architecture or implementation framing.

## Expected Outputs

- user problem summary
- target user list
- product significance summary
- workflow pain-point list

## Exit Criteria

- the PRD has a clear user-facing problem definition
- the target users are explicit
- the value of solving the problem is visible

## Confirmation Gate

Present the user problem and target users, then ask the user for confirmation before proceeding to Step 03.
