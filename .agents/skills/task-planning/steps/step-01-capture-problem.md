# Step 01: Capture Problem

## Purpose

Establish a correct, shared understanding of the task before planning.

## Entry Criteria

- a defined task document exists
- the task links to an HLD, PRD, domain analysis, or other source

## Actions

1. Document the planning version of the problem statement.
2. Identify affected components, systems, modules, runtimes, or boundaries.
3. List initial questions:
   - technical unknowns
   - business rule clarifications
   - edge cases
4. Capture related context from tasks, HLDs, PRDs, and the repository where relevant.

## Expected Outputs

- problem summary
- affected component list
- initial questions
- related context summary

## Exit Criteria

- the problem is clear enough for user confirmation
- affected components are identified
- open questions are explicit

## Confirmation Gate

Present a concise summary and ask permission to proceed to Step 02.
