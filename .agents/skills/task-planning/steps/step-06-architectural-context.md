# Step 06: Architectural Context

## Purpose

Show how the planned work integrates with the surrounding system and where it must be tested.

## Entry Criteria

- Step 05 exit criteria are met

## Actions

1. Create a diagram showing involved components, integration points, runtime boundaries, and test boundaries.
2. Document key relationships between affected components.
3. Highlight where real integration must be validated rather than assumed through mocks.

## Expected Outputs

- architecture diagram
- key relationship summary
- test-boundary summary

## Exit Criteria

- integration points are explicit
- test boundaries are explicit

## Proceed Condition

Proceed to Step 07 when the integration context is clear.
