# Step 04: Update Existing Task with Research Findings

## Purpose

Keep the task definition accurate without overloading it with technical detail.

## Entry Criteria

- Step 03 exit criteria are met
- research findings have been summarized

## Actions

1. Compare the existing task against the research findings.
2. Update the task only if research proves:
   - the problem statement is incorrect
   - goals are materially wrong
   - acceptance criteria are missing or incorrect
3. Do not move technical implementation detail into the task.
4. Put all other findings aside for the technical implementation plan.

## Expected Outputs

- updated task document if required
- explicit note that no task update was required if nothing material changed

## Exit Criteria

- the task remains product-readable
- only corrections to task truth were applied
- technical planning detail is deferred to later steps

## Proceed Condition

Proceed to Step 05 once the task accurately reflects the researched problem and requirements.
